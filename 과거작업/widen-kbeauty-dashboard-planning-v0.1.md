# 위든 K-Beauty 해외판매 운영 대시보드 상세 기획서 v0.2

작성일: 2026-07-04  
대상 프로젝트: `widen_cosmetic`  
대상 사용자: K-Beauty 해외 판매를 운영하는 1인 대표  
초기 타깃 국가: 대만, 일본  
초기 판매 채널: Shopee, Qoo10  
문서 목적: 개발자와 AI 에이전트가 바로 구현 방향을 잡을 수 있도록 제품 목적, 화면 구성, 데이터 구조, 추천 기준, MVP 완료 기준을 구체화한다.

---

## 1. 프로젝트 한 줄 정의

이 대시보드는 K-Beauty 해외 판매자가 매주 “어떤 상품을 고치고, 어떤 제품을 새로 소싱하고, 어떤 경쟁사 움직임을 참고해야 하는지”를 한 화면에서 판단하게 해주는 운영 의사결정 도구다.

단순한 트렌드 모음이나 브랜드 도감이 아니라, 대표가 바로 실행할 수 있는 주간 액션을 만드는 것이 핵심이다.

---

## 2. 왜 필요한가

1인 판매자는 매일 많은 정보를 봐야 한다.

- 올리브영, 무신사, 다이소, SNS에서 뜨는 K-Beauty 제품
- Shopee/Qoo10에서 잘 팔리는 경쟁 상품
- 내 상품의 노출, 클릭, 조회, 찜, 주문, 전환율
- 가격, 쿠폰, 썸네일, 상품명, 상세페이지, 광고 성과
- 대만과 일본 고객 반응 차이

문제는 정보가 부족한 것이 아니라, 정보가 흩어져 있다는 점이다.  
이 대시보드는 흩어진 정보를 모아 “이번 주에 해야 할 일”로 바꿔준다.

---

## 3. 제품의 핵심 원칙

### 3.1 대표는 분석가가 아니라 의사결정자다

대표가 매주 해야 하는 일은 데이터를 직접 해석하는 것이 아니라, AI가 정리한 근거를 보고 채택, 보류, 조사, 실행을 결정하는 것이다.

### 3.2 첫 화면은 보고서가 아니라 실행판이어야 한다

첫 화면의 질문은 다음 하나다.

> 이번 주에 내가 무엇을 해야 하는가?

따라서 첫 화면에는 트렌드 설명보다 액션 항목이 먼저 나와야 한다.

### 3.3 모든 추천에는 이유가 있어야 한다

“추천합니다”만으로는 부족하다. 각 항목에는 최소한 아래 정보가 있어야 한다.

- 왜 추천하는가
- 어떤 데이터를 근거로 삼았는가
- 지금 하지 않으면 어떤 기회를 놓치는가
- 실행하면 어떤 효과를 기대할 수 있는가
- 대표가 지금 내려야 할 결정은 무엇인가

### 3.4 데이터는 누적되어야 한다

매주 새로 보는 화면으로 끝나면 안 된다. 결정 내역과 결과가 쌓여야 한다.

- 어떤 추천을 채택했는지
- 어떤 추천을 보류했는지
- 채택 후 성과가 좋아졌는지
- 어떤 국가와 채널에서 반응이 좋았는지
- 어떤 카테고리가 시즌성인지 반복 트렌드인지

---

## 4. MVP 목표

초기 MVP는 자동 수집의 완성보다 “좋은 운영 화면”을 우선한다.

### 4.1 MVP의 핵심 목표

1. 대표가 이번 주 실행할 액션을 한 화면에서 볼 수 있다.
2. 각 추천에는 근거와 우선순위가 있다.
3. Shopee/Qoo10 상품 개선 후보가 표시된다.
4. 신규 소싱 후보가 표시된다.
5. 경쟁사 변화와 트렌드 신호가 연결된다.
6. 대표가 채택, 보류, 추가 조사, 실행 완료 상태를 남길 수 있다.

### 4.2 MVP에서 하지 않는 것

초기 버전에서는 아래 기능을 무리하게 완성하지 않는다.

- 모든 플랫폼의 완전 자동 크롤링
- 실시간 재고/주문 시스템 연동
- 광고비 자동 집행
- 복잡한 권한 관리
- 다중 사용자 협업 기능
- 정교한 머신러닝 예측 모델

---

## 5. 메인 화면 구조

메인 화면은 “주간 운영 대시보드”로 구성한다.

권장 순서:

1. 이번 주 요약 지표
2. 오늘의 의사결정
3. 이번 주 업데이트할 상품
4. 신규 소싱 후보
5. 경쟁사 변화
6. 대만/일본 트렌드 요약
7. 유입/전환 이상 신호
8. 결정 이력

---

## 6. 이번 주 요약 지표

### 목적

대표가 화면을 열자마자 이번 주 상태를 10초 안에 파악하게 한다.

### 표시 항목

- 업데이트 필요 상품 수
- 신규 소싱 후보 수
- 경쟁사 변화 수
- 트렌드 상승 신호 수
- 전환 문제 상품 수
- 이번 주 미결정 액션 수

### 예시

| 지표 | 값 | 의미 |
| --- | ---: | --- |
| 업데이트 필요 상품 | 5개 | 상품명, 썸네일, 가격, 상세페이지 수정 후보 |
| 신규 소싱 후보 | 4개 | 대만/일본 판매 가능성이 있는 후보 |
| 경쟁사 변화 | 7건 | 가격 변경, 신상품 등록, 프로모션 변경 |
| 전환 문제 상품 | 3개 | 클릭은 있으나 구매가 낮은 상품 |
| 미결정 액션 | 6개 | 대표가 아직 채택/보류하지 않은 항목 |

- 국가
- 채널

- 근거

### 예시

| --- | --- |

| 상태 | 검토 필요 |

## 8. 이번 주 업데이트할 상품

### 목적

이미 판매 중인 Shopee/Qoo10 상품 중 수정하면 성과가 좋아질 가능성이 높은 상품을 찾는다.

### 추천 기준

아래 조건 중 하나 이상에 해당하면 업데이트 후보로 표시한다.

- 노출은 높지만 클릭률이 낮다.
- 클릭은 있으나 주문이 낮다.
- 찜/장바구니는 있으나 구매가 낮다.
- 경쟁사 가격이 낮아졌다.
- 동일 카테고리 트렌드는 상승 중인데 내 상세페이지 설득력이 약하다.
- 썸네일에 핵심 효능, 용량, 가격, 사용 상황이 잘 드러나지 않는다.
- 일본/대만 고객이 이해하기 어려운 상품명이 사용되고 있다.

### 간단한 판단 공식

초기 MVP에서는 아래처럼 단순한 규칙 기반 판단을 사용한다.

| 신호 | 조건 예시 | 추천 액션 |
| --- | --- | --- |
| 노출은 높고 클릭률 낮음 | 노출 1,000 이상, 클릭률 1% 미만 | 썸네일/상품명 개선 |
| 클릭은 있는데 구매 낮음 | 클릭 100 이상, 전환율 1% 미만 | 상세페이지/가격 개선 |
| 찜은 있는데 구매 낮음 | 찜 30 이상, 주문 0~1건 | 쿠폰/프로모션 테스트 |
| 경쟁사 가격 하락 | 경쟁사 평균가가 내 상품보다 10% 이상 낮음 | 가격 조정 또는 구성 변경 |
| 트렌드 상승 | 관련 키워드 신호 3회 이상 증가 | 광고 테스트 또는 상세페이지 보강 |

### 표시 항목

- 상품명
- SKU
- 국가
- 채널
- 현재 성과
- 문제 유형
- 추천 액션
- 근거
- 우선순위
- 예상 효과
- 상태

### 예시

| 상품 | 국가/채널 | 문제 | 추천 액션 | 우선순위 |
| --- | --- | --- | --- | --- |
| VT 리들샷 100 | 대만/Shopee | 노출 대비 클릭률 낮음 | 썸네일 문구와 상품명 수정 | 높음 |
| 아누아 어성초 토너 | 일본/Qoo10 | 장바구니는 있으나 구매 낮음 | 쿠폰 테스트, 배송비 문구 강조 | 높음 |
| 달바 미스트 | 대만/Shopee | 경쟁사 가격 하락 | 2개 묶음 구성 테스트 | 중간 |
| 선크림 A | 일본/Qoo10 | 시즌 상승 대비 상세페이지 약함 | 여름/자외선 키워드 보강 | 중간 |
| 마스크팩 B | 대만/Shopee | 조회수 낮음 | 광고 소액 테스트 | 낮음 |

## 9. 신규 소싱 후보

### 목적

새로 들여올 만한 브랜드/제품을 찾고, 소싱 우선순위를 정한다.

### 평가 기준

| 기준 | 설명 |
| --- | --- |
| 한국 내 트렌드 강도 | 올리브영, 무신사, 다이소, SNS에서 반복적으로 보이는가 |
| 대만 적합도 | 대만 소비자에게 가격, 효능, 사용감, 브랜드 이미지가 맞는가 |
| 일본 적합도 | 일본 고객의 구매 기준과 리뷰 문화에 맞는가 |
| 가격 경쟁력 | 해외 판매가와 원가/배송비를 고려했을 때 마진이 가능한가 |
| 차별성 | 기존 Shopee/Qoo10 상품과 다른 포인트가 있는가 |
| 공급 가능성 | 실제 소싱, 재고 확보, 반복 매입이 가능한가 |
| 상세페이지 난이도 | 번역, 효능 설명, 사용법 설명이 쉬운가 |
| 리스크 | 경쟁 심화, 인증, 통관, 브랜드 정책 문제가 있는가 |

### 점수화 방식

초기 MVP에서는 5점 척도로 평가한다.

- 트렌드 강도: 1~5
- 국가 적합도: 1~5
- 가격/마진 가능성: 1~5
- 차별성: 1~5
- 공급 가능성: 1~5
- 리스크: 낮음/중간/높음

최종 추천은 총점만 보지 않고, 리스크가 높은 상품은 별도로 표시한다.

### 예시

| 후보 | 카테고리 | 추천 국가 | 강점 | 리스크 | 액션 |
| --- | --- | --- | --- | --- | --- |
| 다이소 VT 리들샷 라인 | 앰플/스피큘 | 대만 | 가격 대비 화제성 높음 | 안정적 재고 확보 어려움 | 테스트 소싱 |
| 무신사 상승 남성 쿠션 | 메이크업 | 일본 | 남성 뷰티 관심 증가 | 브랜드 인지도 낮음 | 브랜드 조사 |
| 저자극 선크림 | 선케어 | 일본/대만 | 시즌성 강함 | 경쟁 심함 | 가격 비교 |
| 슬리핑팩 | 마스크팩 | 대만 | 야간 케어 니즈 | 상세 설명 필요 | 상세페이지 샘플 제작 |

## 10. 경쟁사 변화

### 목적

경쟁사가 무엇을 새로 팔고, 어떤 방식으로 판매하는지 추적해 내 운영에 반영한다.

### 수집 대상

- Shopee 대만 내 K-Beauty 경쟁 판매자
- Qoo10 일본 내 K-Beauty 경쟁 판매자
- 특정 브랜드 공식몰 또는 대형 셀러
- 같은 카테고리 베스트셀러 상품 페이지

### 추적 항목

- 신규 등록 상품
- 가격 변경
- 할인/쿠폰/프로모션 변경
- 리뷰 수 증가
- 평점 변화
- 썸네일 변경
- 상품명 변경
- 상세페이지 구성 변경
- 번들 구성 변경

### 대표가 얻어야 하는 결론

- 지금 따라 해볼 만한 판매 포인트는 무엇인가
- 가격을 맞춰야 하는가, 구성을 다르게 해야 하는가
- 상세페이지에서 참고할 문구나 구성은 무엇인가
- 이 경쟁 상품이 단기 유행인지, 지속 가능성이 있는지

## 11. 대만/일본 트렌드 요약

### 목적

대만과 일본에서 판매 가능성이 높은 K-Beauty 흐름을 매주 정리한다.

### 표시 항목

- 상승 중인 카테고리
- 관련 키워드
- 관련 브랜드/제품
- 대만 적합도
- 일본 적합도
- 시즌성 여부
- 스테디셀러 가능성
- 추천 액션

### 트렌드 분류

| 유형 | 설명 | 예시 액션 |
| --- | --- | --- |
| 단기 급등 | SNS나 특정 플랫폼에서 갑자기 뜨는 흐름 | 소량 테스트 |
| 시즌성 | 여름 선케어, 겨울 보습처럼 계절 영향이 큰 흐름 | 상세페이지 시즌 문구 보강 |
| 반복 상승 | 매년 또는 매월 반복되는 수요 | 재고 확보, 광고 테스트 |
| 스테디셀러 | 꾸준히 검색과 구매가 있는 카테고리 | 장기 판매 상품으로 관리 |
| 위험 신호 | 경쟁 과열, 가격 하락, 리뷰 악화 | 보류 또는 가격 전략 재검토 |

## 12. 유입/전환 이상 신호

### 목적

내 상품이 왜 팔리지 않는지 문제 유형을 빠르게 찾는다.

### 문제 유형

| 문제 유형 | 데이터 신호 | 가능한 원인 | 추천 액션 |
| --- | --- | --- | --- |
| 노출 부족 | 노출 수 낮음 | 키워드 부족, 광고 부족 | 상품명 키워드 보강, 광고 테스트 |
| 클릭률 낮음 | 노출 대비 클릭 낮음 | 썸네일 약함, 가격 매력 부족 | 썸네일/가격 표시 개선 |
| 전환율 낮음 | 클릭/조회 대비 주문 낮음 | 상세페이지 설득력 부족 | 상세페이지 보강, 리뷰 강조 |
| 찜 후 구매 낮음 | 찜/장바구니 대비 주문 낮음 | 가격/쿠폰/배송비 문제 | 쿠폰, 묶음 구성 테스트 |
| 국가별 반응 차이 | 대만은 좋고 일본은 낮음 | 국가별 메시지 차이 | 국가별 상품명/상세페이지 분리 |

## 13. AI 직원 역할

대시보드는 내부적으로 여러 AI 직원이 일하는 구조처럼 설계한다.

### 13.1 트렌드 분석 담당

역할:

- 이번 주 뜨는 카테고리와 제품군 정리
- 대만/일본 적합도 평가
- 시즌성, 반복성, 단기 유행 여부 판단

출력:

- 이번 주 트렌드 Top 5
- 국가별 추천 카테고리
- 관련 브랜드/제품
- 추천 액션

### 13.2 경쟁사 분석 담당

역할:

- 경쟁사 신상품, 가격, 프로모션 변화 감지
- 상세페이지와 썸네일 구성 분석
- 참고할 판매 포인트 추출

출력:

- 경쟁사 변화 요약
- 따라 해볼 포인트
- 주의해야 할 가격 변화
- 내 상품에 반영할 액션

### 13.3 소싱 담당

역할:

- 신규 브랜드/제품 후보 추천
- 예상 마진과 리스크 정리
- 소싱 우선순위 제안

출력:

- 신규 소싱 후보 Top 5
- 소싱 문의 우선순위
- 리스크 메모
- 테스트 판매 전략

### 13.4 Shopee/Qoo10 운영 담당

역할:

- 내 상품 성과 이상 신호 감지
- 상품명, 썸네일, 가격, 상세페이지 개선점 제안
- 국가별 반응 차이 분석

출력:

- 이번 주 업데이트 상품 Top 5
- 상품별 문제 원인
- 수정 액션
- 예상 효과

### 13.5 주간 전략 보고 담당

역할:

- 모든 분석을 묶어 대표용 주간 보고 생성
- 이번 주 실행 우선순위 정리

출력:

- 주간 요약
- 이번 주 핵심 판단 5개
- 실행 우선순위
- 다음 주 확인할 지표

## 18. 화면 문구 원칙

대시보드는 보고서처럼 길게 설명하지 않는다.  
대표가 빠르게 판단할 수 있도록 짧고 명확하게 말한다.

좋은 문구:

- “클릭은 있는데 구매가 낮습니다.”
- “가격보다 상세페이지 설득력이 문제일 가능성이 큽니다.”
- “이번 주에는 썸네일 교체를 먼저 테스트하세요.”
- “대만은 반응이 있으나 일본은 보류가 좋습니다.”

피해야 할 문구:

- “시장성이 있어 보입니다.”
- “추후 검토가 필요합니다.”
- “데이터 기반으로 추천됩니다.”
- “브랜드 경쟁력이 우수합니다.”

## 19. MVP 완료 기준

1차 MVP는 아래 조건을 만족하면 완료로 본다.

- 메인 화면에서 이번 주 해야 할 일을 볼 수 있다.
- 오늘의 의사결정 카드가 최소 5개 표시된다.
- 업데이트할 상품 Top 5가 표시된다.
- 신규 소싱 후보 Top 5가 표시된다.
- 경쟁사 변화 요약이 표시된다.
- 대만/일본 트렌드 요약이 표시된다.
- Shopee/Qoo10 유입/전환 문제를 볼 수 있다.
- 각 추천 항목에 이유, 근거, 우선순위, 예상 효과가 있다.
- 대표가 채택, 보류, 추가 조사, 실행 완료 상태를 남길 수 있다.
- 샘플 데이터가 실제 운영 상황처럼 구체적이다.

## 20. 구현 우선순위

### 1단계: 화면 재정렬

현재 화면을 브랜드 피드 중심에서 주간 의사결정 중심으로 바꾼다.

우선 작업:

- 첫 탭을 Dashboard로 변경
- 요약 지표 추가
- 오늘의 의사결정 카드 추가
- 업데이트 상품 Top 5 추가
- 소싱 후보 Top 5 추가

### 2단계: 샘플 데이터 구조화

화면에 직접 박힌 문장을 줄이고, 데이터 객체에서 렌더링하도록 바꾼다.

우선 작업:

- 상품 성과 샘플 데이터
- 경쟁사 변화 샘플 데이터
- 트렌드 신호 샘플 데이터
- 소싱 후보 샘플 데이터
- 액션 추천 샘플 데이터

### 3단계: 결정 상태 저장

대표가 누른 채택/보류/조사/완료 상태를 저장한다.

우선 작업:

- 브라우저 저장소 저장
- 새로고침 후 상태 유지
- 결정 메모 입력
- 결정 이력 표시

### 4단계: 구글 스프레드시트 연결 준비

초기에는 CSV URL을 입력하면 데이터를 불러올 수 있게 한다.

우선 작업:

- Settings 화면에 CSV URL 입력
- CSV 파싱
- 샘플 데이터와 외부 데이터 구조 맞추기
- 연결 실패 시 기본 샘플 데이터 유지

## 21. 이번 단계의 결론

이 프로젝트의 중심은 자동 수집이 아니라 대표의 주간 의사결정이다.

따라서 첫 구현은 다음 한 문장에 집중한다.

> 대만/일본 Shopee/Qoo10 판매를 위해, 이번 주 어떤 상품을 수정하고 어떤 제품을 소싱해야 하는지 한 페이지에서 판단한다.

현재 구현물도 이 방향에 맞춰 브랜드 발견/도감 중심이 아니라 “주간 실행 대시보드” 중심으로 재정렬해야 한다.
