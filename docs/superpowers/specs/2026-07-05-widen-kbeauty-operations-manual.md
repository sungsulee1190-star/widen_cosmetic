# WIDEN K-Beauty SKU Dashboard 운영 매뉴얼 초안

작성일: 2026-07-05  
목적: 대표가 매주 대시보드, Notion, Google Drive를 어떻게 사용해 SKU 기반 해외판매 의사결정을 할지 정리한다.

---

## 1. 운영 철학

이 시스템은 예쁜 보고서를 보기 위한 도구가 아니다. 매주 팔 만한 SKU를 찾고, 경쟁 판매 페이지를 해부하고, 오프라인에서 실제 소싱 가능성을 확인하고, Qoo10/Shopee 판매 전략으로 빠르게 바꾸기 위한 작업판이다.

핵심 질문은 항상 같다.

```text
이 SKU를 팔 것인가?
판다면 어디서 소싱하고, 어느 채널에, 어떤 제목/이미지/페이지로 팔 것인가?
```

---

## 2. 매일 보는 화면: 오늘의 액션

매일 대시보드를 열면 먼저 오늘의 액션을 본다.

### 확인할 것

- 오늘 조사할 SKU
- 오늘 방문할 매장
- 오늘 캡처할 경쟁 판매 페이지
- 오늘 업데이트할 Notion 페이지
- 오늘 작성할 MD 분석 리포트
- 미완료/진행중/보류 작업

### 작업 상태

- 미완료: 아직 시작하지 않음
- 진행중: 조사 중
- 완료: 조사/기록 완료
- 보류: 지금은 판단하지 않음
- 재방문 필요: 오프라인 확인이 더 필요함
- Notion 업데이트 필요: 자료는 있으나 DB 정리가 안 됨
- MD 리포트 작성 필요: 경쟁 판매 페이지 분석을 문서화해야 함

## 4. 월간 루틴

매월 말에는 SKU를 네 가지로 분류한다.

### 4.1 스테디셀러 후보

조건:

- 3개월 이상 반복 노출
- 가격 변동이 크지 않음
- 리뷰/랭킹이 꾸준함
- 오프라인에서도 계속 진열됨

판단:

- 안정적으로 판매 가능한 SKU인지 확인
- 도매가/매입가를 더 깊게 조사
- 상세페이지를 장기 운영용으로 설계

### 4.2 시즌성 후보

조건:

- 특정 계절/행사/세일 기간에 급상승
- 선케어, 쿨링, 마스크팩, 바디, 헤어 등 시즌 영향

판단:

- 다음 시즌 전에 미리 준비할지 결정
- 재고 부담이 큰지 확인

### 4.3 세일 의존 SKU

조건:

- 올리브영 세일, 무신사 할인, 다이소 품절 대란 때만 강함
- 평소 해외 가격차가 약함

판단:

- 세일 기간 단기 판매용으로만 볼지 결정
- 상시 판매는 보류할 수 있음

### 4.4 Fast copycat 후보

조건:

- Qoo10/Shopee에서 잘 파는 경쟁 페이지가 있음
- 제목/이미지/세트 구성 문법이 명확함
- 국내 소싱처가 있음
- 예상 마진이 괜찮음

판단:

- 경쟁 페이지 분석 MD 작성
- 내 판매 페이지 초안 작성
- 테스트 등록 여부 결정

## 5. SKU 조사 절차

### 1단계: SKU 발견

발견 경로:

- 다이소
- 무신사뷰티
- 올리브영
- Qoo10 JP
- Shopee TW
- 인플루언서/SNS
- 경쟁 판매자

### 2단계: SKU 기본 정보 입력

Notion SKU DB에 입력한다.

- SKU명
- 브랜드
- 이미지
- 대분류
- 성분 태그
- 국내 소싱처
- 해외 판매 후보 채널
- 가격
- 발견 경로
- 발견일

### 3단계: 국내 소싱 확인

- 다이소 가격
- 무신사뷰티 가격
- 올리브영 가격
- 세일 여부
- 오프라인 재고
- 대형 매장 진열 여부

### 4단계: 해외 판매 확인

- Qoo10 JP 검색
- Shopee TW 검색
- 경쟁 판매자 수
- 해외 판매가
- 리뷰 수
- 판매량/랭킹
- 쿠폰/세트 구성

### 5단계: Fast copycat 분석

좋은 경쟁 판매 페이지가 있으면 MD 분석 리포트를 작성한다.

- 제목
- 대표 이미지
- 상세페이지
- 가격/쿠폰
- 리뷰 언어
- 따라 할 점
- 변형할 점
- 다음 액션

### 6단계: 판단

상태를 정한다.

- 즉시 테스트
- 오프라인 재확인
- 가격 조사 필요
- 경쟁 페이지 추가 분석
- 보류
- 제외

## 6. 오프라인 방문 매뉴얼

### 6.1 명동

목적:

- 관광객 실제 구매 확인
- 다이소/올리브영 대형 매장 관찰
- 외국어 POP, 택스리펀드, 품절템 확인

체크할 것:

- 관광객이 많이 집는 SKU
- 계산대 근처 소형템
- 선케어/마스크팩/립/패드
- 다이소 저가 뷰티템
- 1+1, 세일, 한정 구성

### 6.2 성수

목적:

- 인디 브랜드와 성분 트렌드 확인
- 팝업/체험형 매장 관찰
- 프리미엄 K-뷰티 표현 방식 참고

체크할 것:

- PDRN
- 시카
- 콜라겐
- 두피/헤어
- 인디 향수
- 브랜드 쇼룸/팝업 표현

### 6.3 홍대

목적:

- 1020, 외국인, 색조, 향수, 남성 그루밍 확인
- 무신사 감성 고객층 관찰

체크할 것:

- 립/틴트
- 쿠션/베이스
- 향수
- 쉐이빙/제모
- 남성 커버 제품
- 여행객 소형템

## 8. Notion 사용법

Notion은 깊은 자료를 쌓는 곳이다.

초기 DB:

1. SKU DB
2. 성분 위키
3. 국가별 판매 주의사항
4. 매장 방문 로그
5. 벤치마킹/레퍼런스 DB

운영 규칙:

- 새 자료는 가능하면 SKU에 연결한다.
- 성분 페이지도 관련 SKU를 연결한다.
- 매장 방문 로그도 확인한 SKU를 연결한다.
- 경쟁 판매 페이지도 관련 SKU를 연결한다.
- 국가별 주의사항도 영향을 받는 SKU를 연결한다.

```text
WIDEN_KBeauty_Research
```


## 12. 운영 결론

이 시스템은 매주 쌓일수록 강해진다. 처음부터 모든 자동화를 만들 필요는 없다. 중요한 것은 SKU 하나를 발견했을 때, 그 SKU에 이미지, 가격, 소싱처, 경쟁 페이지, 성분, 국가 리스크, 오프라인 방문 기록, MD 분석 리포트가 계속 연결되는 구조를 만드는 것이다.

대표가 매주 해야 할 일은 데이터를 직접 흩어져서 보는 것이 아니라, 대시보드에서 오늘의 액션을 확인하고, Notion과 Drive에 쌓인 근거를 바탕으로 판매/보류/추가조사를 결정하는 것이다.
