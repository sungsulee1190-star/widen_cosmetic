# WIDEN K-Beauty SKU Intelligence Dashboard 설계 초안

작성일: 2026-07-05  
프로젝트: WIDEN K-Beauty 해외판매 운영 대시보드  
상태: 브레인스토밍 기반 설계 초안  
목적: 기존 대시보드를 처음부터 다시 설계하기 위한 실행 기획서 + 개발 명세서

---

## 1. 한 줄 정의

WIDEN K-Beauty SKU Intelligence Dashboard는 K-Beauty 해외 판매자가 SKU 단위로 인기, 가격, 경쟁 판매 방식, 오프라인 소싱 가능성, 성분/국가별 리스크를 누적 추적하고, Qoo10/Shopee 판매 전략으로 빠르게 전환하기 위한 운영 의사결정 시스템이다.

기존 대시보드의 문제는 정보가 짧고 얕으며, 실제 판매 결정을 내릴 만큼 이미지, 링크, 가격, 경쟁 페이지, 방문 기록, 근거가 부족했다는 점이다. 새 구조는 "보기 좋은 요약 화면"이 아니라 "이번 주 무엇을 조사하고, 무엇을 팔고, 어떤 판매 페이지를 참고할지"를 결정하는 조종석을 목표로 한다.

---

## 2. 핵심 원칙

### 2.1 중심축은 SKU다

모든 데이터는 SKU에 연결된다.

- 브랜드는 SKU를 이해하기 위한 상위 맥락이다.
- 성분은 SKU의 판매 포인트와 규제 리스크를 판단하기 위한 태그다.
- 매장 방문 기록은 SKU의 소싱 가능성을 확인하기 위한 증거다.
- 경쟁사/인플루언서/판매 페이지는 SKU를 어떻게 팔지 판단하기 위한 레퍼런스다.
- 국가별 규제/주의사항은 SKU를 일본/대만에 팔 때 사용할 수 있는 표현과 피해야 할 표현을 판단하기 위한 기준이다.

### 2.2 현재 인기보다 누적 흐름이 중요하다

대시보드는 단발성 인기 상품 목록이 아니라, 주간/월간/누적 기록을 통해 스테디셀러, 시즌성, 세일 의존 SKU, 반복 노출 SKU를 구분해야 한다.

필수 기간 필터:

- 이번 주
- 이번 달
- 최근 3개월
- 최근 6개월
- 누적

### 2.3 Fast copycat이 핵심 판매 전략이다

Fast copycat은 단순 복제가 아니라, 잘 팔리는 판매 페이지의 구조를 빠르게 해부하고 내 상품 페이지 전략으로 변환하는 방식이다.

수집해야 할 항목:

- 제목 구조
- 대표 이미지 구성
- 상세페이지 첫 화면 메시지
- 가격/쿠폰/세트 구성
- 리뷰에서 반복되는 구매 이유
- 고객 문제를 표현하는 언어
- 내가 바로 따라 할 것
- 변형해서 쓸 것
- 피해야 할 것

### 2.4 웹 대시보드는 조종석, Notion/Drive는 저장소다

웹 대시보드는 매주 보는 판단 화면이다. 깊은 자료는 Notion과 Google Drive에 누적한다.

- 웹 대시보드: 요약, 필터, 체크, 우선순위, 링크
- Notion: SKU DB, 성분 위키, 국가별 주의사항, 방문 로그, 벤치마킹 샵 DB
- Google Drive: MD 리포트, 이미지 캡처, 판매 페이지 스크린샷, 상세 분석 원본 자료

## 4. 레이아웃

### 4.1 기본 구조

왼쪽 접이식 사이드바와 오른쪽 메인 작업 화면으로 구성한다.

사이드바는 펼쳤을 때 메뉴명과 미결정 숫자를 보여주고, 접었을 때는 아이콘만 남긴다.

예시:

```text
WIDEN
[접기/펼치기]

오늘의 액션              7
SKU 인텔리전스           42
Qoo10 Copycat 보드        12
성분 위키                18
국가별 주의사항           2
오프라인 소싱 루트        9
경쟁사/인플루언서         25
Notion/Drive DB
설정
```

### 4.2 디자인 방향

기존 대시보드처럼 장식적인 카드 중심이 아니라, 운영자가 반복해서 쓰기 좋은 업무형 화면을 지향한다.

요구사항:

- 이미지를 충분히 보여준다.
- 링크가 명확하게 노출된다.
- 텍스트는 짧은 요약이 아니라 판단에 필요한 근거 중심으로 쓴다.
- 카드마다 다음 액션이 있어야 한다.
- 버튼/체크박스/상태값이 있어야 한다.
- 모바일보다 데스크톱 운영 화면을 우선한다.

## 5. 메뉴별 상세 설계

## 5.1 오늘의 액션

첫 화면이다. 사용자가 열자마자 "오늘 무엇을 해야 하는지"를 알 수 있어야 한다.

### 핵심 기능

- 오늘 할 일 체크
- SKU별 조사 상태 확인
- 방문할 매장 체크
- Qoo10/Shopee 경쟁 페이지 분석 할당
- 미결정 액션 상태 변경

### 카드 예시

```text
작업: Qoo10 JP 경쟁 판매 페이지 5개 캡처
대상 SKU: VT 리들샷 100
채널: Qoo10 JP
목적: 제목/대표 이미지/세트 구성 분석
상태: 미완료
연결: SKU 페이지, Qoo10 검색 링크, Drive 폴더
액션: 진행중 / 완료 / 보류 / 재조사
```

### 상태값

- 미완료
- 진행중
- 완료
- 보류
- 재방문 필요
- Notion 업데이트 필요
- MD 리포트 작성 필요

### 대분류 카테고리

초기에는 대분류만 사용한다.

- 전체
- 향수
- 립
- 베이스
- 아이메이크업
- 스킨케어
- 선케어
- 클렌징/필링
- 바디케어
- 헤어케어
- 디바이스
- 헬스/푸드
- 쉐이빙/제모

### 필터

- 기간: 이번 주, 이번 달, 3개월, 6개월, 누적
- 국내 채널: 다이소, 무신사뷰티, 올리브영

- 해외 판매처 링크

- 최근 가격

## 5.4 성분 위키

성분 위키는 연구 백과사전이 아니라 판매 판단과 마케팅 표현을 위한 위키다.

### 목적

- 이 성분이 들어간 SKU를 팔 만한지 판단한다.
- 일본/대만 판매 시 사용할 수 있는 표현과 조심해야 할 표현을 구분한다.
- 상세페이지/상품명/SNS 카피에 활용할 수 있는 표현을 쌓는다.

### 초기 성분 후보

- PDRN
- 리들샷/스피큘
- 시카
- 어성초
- 레티놀
- 콜라겐
- TXA
- 나이아신아마이드
- 글루타치온
- 세라마이드
- 아젤라인산
- 히알루론산

### 성분 페이지 항목

- 성분 설명
- 왜 요즘 뜨는가
- 어떤 고객이 찾는가
- 대표 SKU
- 관련 브랜드

- 대만 판매 적합성
- 사용할 수 있는 마케팅 표현
- 조심해야 할 표현
- 관련 Qoo10/Shopee 검색 링크
- 관련 인플루언서/콘텐츠 링크

## 5.5 국가별 주의사항

일본과 대만을 우선한다.

### 목적

### 국가 페이지 항목

- 국가
- 화장품 판매 주의사항
- 금지/주의 성분
- 조심해야 할 효능 표현
- 의약품처럼 보일 수 있는 표현
- 라벨/성분표 체크
- 통관/수입 주의사항
- 플랫폼별 주의사항
- 판매 전 체크리스트
- 관련 SKU

## 5.6 오프라인 소싱 루트

방문 권역은 명동, 성수, 홍대를 우선한다. 사무실 위치가 을지로3가역 인근이므로 명동은 평일 짧은 정찰 루트로 활용하기 좋다.

### 권역별 목적

명동:

- 관광객 실구매 관찰
- 다이소/올리브영 대형 매장 방문
- 택스리펀드, 품절, 외국어 POP, 세일 상품 확인

성수:

- 올리브영N 성수
- 인디 브랜드, 팝업, 성분 트렌드, MZ 반응 확인
- 프리미엄/체험형 진열 참고

홍대:

- 무신사 감성 고객층
- 1020, 외국인, 색조, 향수, 남성 그루밍 관찰
- 올리브영/다이소 홍대권 보조 정찰

### 방문 로그 항목

방문 기록도 SKU 기준으로 연결한다.

- 방문일
- 권역
- 매장명
- 확인 SKU
- 가격
- 재고
- 진열 위치
- 세일/쿠폰
- 관광객 반응
- 사진/캡처 링크
- 다음 액션

## 6. 데이터 저장 구조

### 6.1 웹 대시보드

대시보드는 모든 자료를 직접 저장하지 않는다. 요약, 필터, 체크, 링크 역할을 한다.

저장할 수 있는 최소 로컬 상태:

- 오늘의 액션 체크 상태
- 최근 본 필터
- 즐겨찾기 SKU

### 6.2 Notion DB

초기 Notion DB는 5개로 나눈다.

1. SKU DB
2. 성분 위키 DB
3. 국가별 주의사항 DB
4. 매장 방문 로그 DB
5. 벤치마킹/레퍼런스 DB

모든 DB는 SKU와 연결될 수 있어야 한다.

### 6.3 Google Drive

Drive는 원본 자료 보관소다.

저장 대상:

- 경쟁 판매 페이지 MD 분석 리포트
- 판매 페이지 스크린샷
- 대표 이미지 캡처
- 상세페이지 캡처
- 매장 방문 사진
- SKU별 자료 폴더

## 10. 설계 결론

새 대시보드는 브랜드 소개 페이지가 아니다. SKU 단위로 자료를 쌓고, 인기 흐름을 보고, 오프라인 소싱 가능성을 확인하고, Qoo10/Shopee 경쟁 판매 방식을 분석해서 빠르게 내 판매 전략으로 바꾸는 운영 시스템이다.

가장 중요한 성공 기준은 다음이다.

- 사용자가 오늘 해야 할 일을 체크할 수 있다.
- SKU별로 주간/월간/누적 인기 흐름을 볼 수 있다.
- Qoo10 경쟁 판매 페이지를 보고 Fast copycat 전략으로 변환할 수 있다.
- 성분/국가별 주의사항이 판매 판단에 연결된다.
- Notion과 Google Drive에 자료가 계속 쌓인다.
- 화면에 이미지, 링크, 가격, 근거, 다음 액션이 충분히 들어간다.
