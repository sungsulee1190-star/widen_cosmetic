# 위든 K-Beauty 해외판매 운영 대시보드 기획서 v0.1

작성일: 2026-07-04  
대상 프로젝트: `widen_cosmetic`  
구현 도구: Antigravity 2.0  
검토/기획 역할: Codex

## 1. 프로젝트 목적

이 프로젝트는 1인 기업가가 Shopee와 Qoo10에서 대만/일본 고객에게 K-Beauty 제품을 판매할 때, 시장 조사와 운영 판단을 자동화하기 위한 대시보드다.

핵심 목표는 단순한 데이터 수집이 아니라, 대표가 매주 중요한 판단에 집중할 수 있게 만드는 것이다.

- 한국 K-Beauty 트렌드, 경쟁사 변화, 브랜드/제품 정보를 계속 수집한다.
- Shopee/Qoo10의 내 상품 성과와 연결해 판매 개선 포인트를 찾는다.
- 매주 업데이트할 상품, 신규 소싱 후보, 가격/상세페이지/광고 액션을 추천한다.
- 데이터는 매일 누적되어 월별, 시즌별, 국가별 흐름을 볼 수 있어야 한다.

## 2. 운영 원칙

대표는 가능한 한 손을 적게 대고, 중요한 의사결정만 한다.

시스템은 아래 질문에 매주 답해야 한다.

- 지금 대만/일본에서 팔 가능성이 높은 K-Beauty 제품군은 무엇인가?
- 경쟁사들은 어떤 상품을 새로 올리고, 어떤 가격/프로모션으로 판매하는가?
- 내 Shopee/Qoo10 상품 중 이번 주에 수정해야 할 상품은 무엇인가?
- 새로 소싱 검토할 브랜드/제품은 무엇인가?
- 유입은 있는데 구매가 안 되는 상품은 무엇인가?
- 상품명, 썸네일, 상세페이지, 가격, 광고 중 무엇을 고쳐야 하는가?

## 3. 초기 타깃 시장과 판매 채널

### 타깃 국가

- 대만
- 일본

### 판매 채널

- Shopee

초기 설계는 대만과 일본을 우선으로 한다. 이후 성과가 쌓이면 동남아 다른 국가나 추가 채널로 확장할 수 있다.

## 4. MVP 방향

초기 MVP는 데이터 수집 범위를 무리하게 넓히기보다, 대표가 바로 판단할 수 있는 1페이지 운영 대시보드를 우선 완성한다.

추천 방향은 `운영 대시보드 중심 MVP`다.

우선순위:

1. 메인 1페이지 대시보드
2. 필요한 데이터 항목 정의
3. AI 직원 역할별 출력물 정의
4. 경쟁사/트렌드/내 쇼핑몰 성과 데이터 누적 구조
5. 주간 액션 추천

## 5. 메인 1페이지 대시보드 구성

첫 화면은 `이번 주에 내가 무엇을 해야 하는가?`에 답해야 한다.

### 5.1 이번 주 트렌드 요약

목적: 대만/일본 판매 가능성이 높은 K-Beauty 흐름을 빠르게 보여준다.

표시 항목:

- 상승 중인 카테고리
- 상승 근거
- 관련 브랜드/제품
- 대만 적합도
- 일본 적합도
- 시즌성 여부
- 스테디셀러 가능성

예시 카테고리:

- 선케어
- 패드
- 앰플
- 클렌저
- 마스크팩
- 슬리핑팩
- 저자극/민감성 제품

### 5.2 이번 주 업데이트할 상품

목적: 이미 Shopee/Qoo10에 등록된 상품 중 수정이 필요한 상품을 보여준다.

추천 기준:

- 조회수는 높지만 구매 전환이 낮음
- 노출은 있지만 클릭률이 낮음
- 장바구니/찜은 있지만 구매가 낮음
- 경쟁사 가격이 변동됨
- 트렌드가 상승 중인데 내 상품 상세페이지가 약함
- 썸네일, 상품명, 가격, 상세페이지 개선 여지가 큼

표시 항목:

- 상품명
- 판매 채널
- 국가
- 현재 문제
- 추천 액션
- 우선순위
- 예상 효과

추천 액션 예시:

- 상품명 수정
- 썸네일 교체
- 가격 조정
- 상세페이지 개선
- 쿠폰/프로모션 테스트
- 광고 테스트
- 보류

### 5.3 신규 소싱 후보

목적: 새로 들여올 만한 브랜드/제품을 추천한다.

표시 항목:

- 브랜드명
- 제품명
- 카테고리
- 한국 내 인기 근거
- 대만/일본 판매 가능성
- 예상 가격대
- 예상 마진 가능성
- 차별점
- 리스크
- 추천 소싱 액션

리스크 예시:

- 경쟁이 심함
- 가격 경쟁력이 낮음
- 브랜드 인지도가 낮음
- 해외 배송/통관 이슈 가능성
- 상세페이지 번역 난이도 높음

### 5.4 경쟁사 변화

목적: 경쟁사들이 무엇을 새로 팔고, 어떻게 판매하는지 추적한다.

수집 대상:

- 경쟁사 쇼핑몰 3~4개
- 경쟁사 제품 페이지
- 경쟁사 브랜드/카테고리 페이지

표시 항목:

- 경쟁사명
- 신규 등록 상품
- 가격 변화
- 프로모션 변화
- 리뷰 증가 상품
- 상세페이지 구성 변화
- 참고할 만한 판매 포인트

### 5.5 내 쇼핑몰 유입/전환 신호

목적: Shopee/Qoo10에서 내 상품의 유입과 전환 문제를 찾는다.

표시 항목:

- 노출 수
- 클릭 수
- 클릭률
- 조회 수
- 찜/장바구니
- 주문 수
- 전환율
- 국가
- 채널
- 문제 유형

문제 유형:

- 노출 부족
- 클릭률 낮음
- 클릭은 있으나 구매 낮음
- 장바구니/찜은 있으나 구매 낮음
- 가격 경쟁력 부족
- 상세페이지 설득력 부족

### 5.6 오늘의 의사결정

목적: 대표가 바로 판단해야 할 일을 한 곳에 모은다.

의사결정 유형:

- 채택
- 보류
- 추가 조사
- 소싱 문의
- 상품 업데이트
- 광고 테스트
- 가격 조정

화면에서는 액션별로 `왜 이 결정을 해야 하는지` 근거가 함께 보여야 한다.

## 7. AI 직원 역할

대시보드는 내부적으로 여러 AI 직원 역할을 가진 것처럼 설계한다.

### 7.1 트렌드 분석 담당

역할:

- 이번 주 뜨는 제품군 정리
- 시즌성/반복성 판단
- 대만/일본 판매 가능성 평가
- 트렌드 요약 생성

출력물:

- 이번 주 트렌드 Top 5
- 국가별 추천 카테고리
- 시즌성 판단
- 스테디셀러 후보

### 7.2 경쟁사 분석 담당

역할:

- 경쟁사 신상품 감지
- 가격/프로모션 변화 감지
- 상세페이지 구조 분석
- 잘 팔릴 가능성이 있는 패턴 추출

출력물:

- 경쟁사 변화 요약
- 참고할 판매 포인트
- 가격 비교
- 상세페이지 개선 힌트

### 7.3 소싱 담당

역할:

- 소싱 후보 브랜드/제품 추천
- 예상 마진 가능성 정리
- 차별점과 리스크 정리
- 소싱 우선순위 제안

출력물:

- 신규 소싱 후보 Top 5
- 소싱 문의 우선순위
- 리스크 메모
- 예상 판매 전략

### 7.4 Shopee/Qoo10 운영 담당

역할:

- 내 상품 성과 이상 신호 감지
- 상품명/썸네일/가격/상세페이지 개선 포인트 제안
- 국가별 반응 차이 분석

출력물:

- 이번 주 업데이트 상품 Top 5
- 상품별 문제 원인
- 수정 액션 추천
- 예상 효과

### 7.5 주간 전략 보고 담당

역할:

- 모든 데이터를 합쳐 대표가 볼 주간 보고서 생성
- 이번 주 실행 우선순위 제시

출력물:

- 주간 요약
- 이번 주 핵심 판단 5개
- 소싱 후보
- 상품 업데이트 후보
- 위험 신호

## 8. 데이터 흐름

기본 흐름:

1. 외부 데이터 수집
2. 원본 데이터 저장
3. 제품/브랜드/경쟁사 단위로 정리
4. AI 분석 생성
5. 대시보드 카드와 표에 반영
6. 대표가 채택/보류/조사/실행 결정
7. 결정과 결과가 다시 누적됨

중요한 점:

- 원본 데이터와 AI 분석 결과를 분리해서 저장한다.
- AI 분석은 나중에 다시 생성할 수 있어야 한다.
- 대표의 의사결정 기록도 데이터로 남긴다.
- 시간이 지날수록 월별/시즌별/국가별 패턴을 볼 수 있어야 한다.

## 12. Antigravity 구현 요청 기준

Antigravity에서 구현할 때는 아래 순서로 진행하는 것을 권장한다.

1. 대시보드 UI 뼈대 구현
2. 샘플 데이터 기반 메인 1페이지 완성
3. 데이터 모델/저장 구조 설계
4. 경쟁사/트렌드/판매채널 수집 모듈을 붙일 수 있는 인터페이스 준비
5. AI 분석 결과를 저장하고 화면에 표시하는 구조 구현
6. 실제 데이터 수집 자동화는 단계적으로 연결

초기에는 실제 크롤링보다 `샘플 데이터 + 명확한 데이터 구조 + 좋은 의사결정 화면`이 더 중요하다.

## 13. MVP 완료 기준

1차 MVP는 아래 조건을 만족하면 완료로 본다.

- 메인 1페이지에서 이번 주 해야 할 일을 볼 수 있다.
- 업데이트할 상품 Top 5가 표시된다.
- 신규 소싱 후보 Top 5가 표시된다.
- 경쟁사 변화 요약이 표시된다.
- 대만/일본 트렌드 요약이 표시된다.
- Shopee/Qoo10 유입/전환 문제를 볼 수 있다.
- 각 추천 항목에 이유와 우선순위가 있다.
- 대표가 채택/보류/조사/실행 상태를 남길 수 있다.
